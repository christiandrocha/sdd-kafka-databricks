## Summary

Consolidated Bronze, Silver, `silver_users`, Gold, and Quarantine — 37 Unity Catalog tables
total — into one Lakeflow Declarative Pipeline (`pipelines/ubereats_pipeline.py`, renamed
from `bronze_silver_dlt.py`), shared identically by all 3 DABs targets (`dev`, `prod`,
`free_edition`). Before this feature, the same conceptual pipeline existed as three
structurally different implementations: a hybrid Lakeflow-pipeline-plus-7-notebook-task job
for `dev`/`prod`, and a fully separate 37-task classic notebook job for `free_edition`. All
8 legacy notebooks (`pipeline_bronze.ipynb`, `pipeline_silver.ipynb`, `pipeline_users.ipynb`,
6× `cross_domain/gold_*.ipynb`) are retired, with their logic ported verbatim into 8 new
Python registration functions. `databricks.yml` shrank from 878 to 153 lines — the 37-task
YAML anchor scaffold that existed purely to keep `free_edition` symmetric with a job
structure `dev`/`prod` no longer fully used is gone, replaced by one shared
`pipeline_resource`/`pipeline_task` anchor pair used identically by all 3 targets. A new
`docs/adr/007_pipeline_unification.md` documents the reversal of `ADR-006`'s "Explicitly NOT
migrated" section. The code/IaC work is complete and locally verified (lint, 196 pre-existing
tests, bundle YAML resolution); the 4 acceptance tests require a live Databricks workspace
deploy not available in this session, and are recorded below as a tracked follow-up rather
than blocking the ship.

## Lessons Learned

### Process

- Reading every source artifact in full before designing (all 6 Gold notebooks, the `silver_users` notebook, and the entire 878-line `databricks.yml`) paid off directly — DESIGN's code patterns were verbatim-portable into Build with zero rework, because no detail had to be re-derived or guessed mid-build.
- Running all 5 SDD phases (`brainstorm` → `define` → `design` → `build` → `ship`) back-to-back in one session for a feature this well-scoped meant each phase's grounding work carried forward cleanly — DEFINE's clarity score (14/15) and DESIGN's 7 inline ADRs both stayed accurate all the way through Build without needing an `/iterate` correction.

### Technical

- A `@dp.table` decorator's behavior (incremental streaming table vs. fully-recomputed materialized view) is determined by whether the function body returns a streaming or static DataFrame — this let one `register_bronze()` function serve both `kafka` and `volume` source modes without a second registration path, and let Gold/`silver_users` express "full batch recompute" (matching their actual prior behavior) without forcing them into `create_streaming_table`/`create_auto_cdc_flow`.
- DESIGN's own pseudocode for `silver_users`'s quarantine (reading back `dp.read(quarantine_table)` inside `silver.users`'s function) turned out to be more complex than necessary — both quarantine causes (`missing_cpf`, `duplicate_user_id`) reduce to one boolean predicate per candidate row, so a single shared temp view with a nullable `_quarantine_reason` column was simpler and avoided an unnecessary cross-table read-back. Worth trusting "can this be one predicate?" before reaching for the cross-table-read pattern the generic Silver domains need for a genuinely different reason (their uniqueness check needs prior Silver state, `silver_users` doesn't).
- Caught a real latent bug while implementing Decision 2 (`source_mode` branch): an unconditional `_avro_schema_str(kafka_topic)` call would have made Schema Registry reachability a hard requirement even in `volume` mode — defeating the purpose of `volume` mode, which exists specifically because Free Edition serverless may not reach *either* Kafka or its Schema Registry (same network boundary). Gating the call inside the `kafka`-only branch fixed this before it could surface as a confusing deploy-time failure.

### Communication

- Surfacing the CI gap (`bundle-validate` only checked `--target dev`) during DESIGN rather than treating it as out-of-scope avoided shipping a feature where the most-changed target (`free_edition`, 37 tasks → 1) had zero CI coverage of its own.
- Flagging `scripts/preflight_unity_catalog.sh`'s now-unused `checkpoints` Volumes as a known, explicitly out-of-scope consequence (rather than silently leaving it for someone to discover later) keeps the decision auditable without scope-creeping this feature into infrastructure cleanup it didn't ask for.

### Tools & Libraries

- `databricks bundle validate` is useful even without live credentials — it resolves variables/anchors/target overrides fully before attempting authentication, so "does the YAML structure actually merge correctly across 3 targets" was verifiable locally despite an expired token blocking full schema validation.
- `ruff`'s per-path `exclude` config (already excluding `pipelines/` because `spark` is a Databricks-injected global, not a real import) meant no lint noise had to be suppressed inline — worth checking existing tool config for established exceptions before assuming a new file needs special-casing.
