## Lessons Learned

### What worked

- **Agent delegation pattern**: splitting Bronze / Silver / Gold / DABs / CI-CD / Observability across separate agents kept each session focused. No agent had to hold more than one layer in context.
- **Contracts-first**: building Agent 4 (data-contracts) before notebooks meant Silver/Gold had validated DDL generators from day one — no schema guessing in notebooks.
- **Read before edit**: consistently reading existing files before modifying them (ci.yml, deploy.yml, Makefile) prevented full rewrites and preserved prior work.
- **`ruff check . --fix`**: auto-fix resolved 7 violations in `load_to_postgres.py` instantly — no manual edit required.
- **`workflow_run` trigger on deploy.yml**: ensures prod deploys only fire after CI passes on `main`; much safer than direct `push: branches: [main]` which runs CI and deploy in parallel.

### What did not work / surprises

- **ruff linting `.ipynb` notebooks**: ruff 0.15 automatically lints `.ipynb` files and flags Databricks runtime globals (`spark`, `dbutils`) as F821. This was unexpected — `exclude = ["notebooks"]` is the correct fix for Databricks projects.
- **`payment_current_state` phantom**: listed as Silver domain in CLAUDE.md but has no contract YAML, no Bronze source, and no Gold consumer. Caused DABs task count discrepancy (37 vs 39). Design assumption not backed by implementation.
- **NotebookEdit required for `.ipynb`**: the standard `Edit` tool rejects notebook files. Agent 7 had to use `ToolSearch` to load the `NotebookEdit` schema before patching `pipeline_users.ipynb` for `user_id`.
- **`or vector(0)` in Grafana queries**: needed for the "Failed Tasks" stat panel to show 0 instead of "No data" when all connectors are healthy — a Prometheus/Grafana gotcha that only appears in practice.
